BITUpdateManagerDelegate

@protocol BITUpdateManagerDelegate <NSObject>

The BITUpdateManagerDelegate formal protocol defines methods further configuring the behaviour of BITUpdateManager.

Update
  • -shouldDisplayUpdateAlertForUpdateManager:forShortVersion:forVersion:

    Return if update alert should be shown

    If you want to display your own user interface when there is an update available, implement this method, present your user interface and return NO. In this case it is your responsibility to call BITUpdateManager showUpdateView

    Note: This delegate will be invoked on startup and every time the app becomes active again!

    When returning YES the default blocking UI will be shown.

    When running the app from the App Store, this delegate is ignored.

Expiry
  • -shouldDisplayExpiryAlertForUpdateManager:

    Return if expiry alert should be shown if date is reached

    If you want to display your own user interface when the expiry date is reached, implement this method, present your user interface and return NO. In this case it is your responsibility to make the app unusable!

    Note: This delegate will be invoked on startup and every time the app becomes active again!

    When returning YES the default blocking UI will be shown.

    When running the app from the App Store, this delegate is ignored.

Privacy
  • -updateManagerShouldSendUsageData:

    Return NO if usage data should not be send

    The update module send usage data by default, if the application is NOT running in an App Store version. Implement this delegate and return NO if you want to disable this.

    If you intend to implement a user setting to let them enable or disable sending usage data, this delegate should be used to return that value.

    Usage data contains the following information:

  • App Version
  • iOS Version
  • Device type
  • Language
  • Installation timestamp

  • Usage time

    • Warning

    When setting this to NO, you will NOT know if this user is actually testing!

Privacy
  • -willStartDownloadAndUpdate:

    Implement this method to be notified before an update starts.

    The update manager will send this delegate message just before the system call to update the application is placed, but after the user has already chosen to install the update.

    There is no guarantee that the update will actually start after this delegate message is sent.

    updateManager

  • -updateManagerWillExitApp:

    Invoked right before the app will exit to allow app update to start (>= iOS8 only)

    The iOS installation mechanism only starts if the app the should be updated is currently not running. On all iOS versions up to iOS 7, the system did automatically exit the app in these cases. Since iOS 8 this isn’t done any longer.
